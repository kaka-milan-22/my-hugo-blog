> 如果 Web3 是真正的去中心化，那现在就是 **Web 2.5**——这个说法比"Web3 来了！"或"Web3 是骗局"都更诚实，也是目前行业里最有解释力的中间态描述。

过去几年，关于区块链的讨论被两个极端占据：一方坚信 Web3 正在接管世界、金融和身份都会被重塑；另一方认为整个加密行业就是庞氏骗局，毫无实质意义。两边都不准确。

真正发生的事情是：**区块链在某些维度实现了 Web3，在更多维度还停留在 Web2。** 我们正处在 Web 2.5 的时代。这篇文章做两件事：

1. 把 Web 2.5 这个框架讲透——它**真实**添加了哪些 Web 2 永远做不到的东西，又在哪些地方**还没突破**。
2. 用一套判断标准（**谁能改协议 / 谁能冻资产 / 出事能否自救**），把当下主流加密产品逐一归位到 Web 2、Web 2.5、Web 3 三层。

看完这篇文章，你再去看任何新出现的加密产品，都能迅速判断它在堆栈的哪一层，从而给风险正确定价。

## 一、Web 2.5 真实加了什么，Web 2 永远做不到

### 1.1 资产层的所有权结构换了

你钱包里的 USDT、ETH、NFT、LP 代币——这些不是某家公司数据库里的一条记录，而是一条**公开账本上你控制的私钥所签名的条目**。

举个具体的对比：

| 场景 | Web 2 (支付宝/银行) | Web 2.5 (钱包自托管) |
|------|---------------------|----------------------|
| 平台倒闭 | 资产被冻结，排队等清算 | 币还在链上，换个钱包软件继续用 |
| 平台锁你账户 | 申诉可能等数月，甚至永远拿不回来 | 无人能锁，私钥在你手里 |
| 跨国转账 | 需要 SWIFT、几天到账、收手续费 | 几秒到账，手续费链上透明 |

这一层改变本身就是**代际跨越**——在 Web 2 金融里，只有实物黄金能类比。并且**这个改变不因为前端是中心化的就打折**：只要你把币提到自己钱包，Coinbase 即使明天倒闭，你的资产也分毫不动。

### 1.2 身份和声誉可移植

一个 `vitalik.eth` 名字跨所有 dApp 通用。

而在 Web 2 里：

- Instagram 账号迁不到 TikTok
- 微信数据出不了腾讯
- 你的 LinkedIn 关系网在 LinkedIn 死掉那天就消失了

钱包地址作为身份虽然糙（一串 42 位十六进制），但它**归你，不归任何平台**。更重要的是，DID、Proof of Humanity、Farcaster 这些实验正在把这层做厚。未来五年可能会有真东西落地。

### 1.3 应用的可组合性是原子化的

这是很多人没意识到的**商业协作模式**的根本变革。

在 Web 2 里：Uber 想接 Booking 的酒店 API，要谈 6 个月商务合同，再走 12 个月 API 对接，每次调用可能还要付钱。

在 Web 2.5 里：**任何合约无许可调用任何合约**。

- Yearn 可以堆叠 Curve 做收益聚合器，一行代码
- Aave 可以调 Uniswap 做清算，一行代码
- 一个独立开发者一小时就能部署一个协议，嵌入十个已有协议的能力

这不是 UI 层的差别，这是**整个商业协作模式**的差别——从"先签合同再对接"变成"直接调用"。Uniswap v3 被调用了上万次，没收过一分"商务合作费"。

### 1.4 状态全球透明

Tether 的全部余额表、Uniswap 的流动性分布、任何地址的历史——**免费、实时、不可删除**。

Web 2 的"透明报告"和这个差十个数量级：

- 银行给你的月度对账单是精选过的摘要
- 上市公司季报一季度一次、延迟 45 天
- "Twitter 透明度报告"是 PDF，不是实时接口

链上数据倒逼出的市场效率（套利、风控、链上分析）正在重塑金融基础设施。连美联储都在研究以太坊数据做宏观分析——这不是观点，是他们公开发表的论文。

### 1.5 退出权是硬的

不管协议多糟糕，你**都能提币走人**（只要不被黑名单，而绝大多数链上产品没有黑名单）。

Web 2 里 Facebook 锁你账号你毫无办法——你的朋友、照片、回忆都在它手里。**这个退出权本身就是用户权力的根本转换。**

---

> 以上五条加起来，就是从 Web 2 到 Web 2.5 的 **0.5 分进展**。关键是：**这个进展不可逆。**
> 一旦自托管成为可能、可组合性被证明、链上透明度被习惯，传统金融也必须向这些方向靠拢。RWA 代币化、CBDC 设计、J.P. Morgan 的 Onyx 链——都是在抄 Web 2.5 的作业。

## 二、Web 2.5 还没到 Web 3 的地方

但我们也必须同样诚实地承认——Web 2.5 远不是 Web 3。以下五点是"剩下的 0.5 分"。

### 2.1 基础设施 90% 还跑在 AWS 上

以太坊节点的前三大运营商——Coinbase、Lido、Binance——的物理服务器大部分在 AWS、GCP、Hetzner。

> **去中心化协议跑在中心化云上**，一次 AWS us-east-1 故障能让以太坊出块延迟明显。
>
> 协议去中心化 + 物理设施中心化 = **脆弱**。

### 2.2 RPC 层是单点

Infura + Alchemy 占以太坊 RPC 流量的 90%。这两家都是美国实体，**能也愿意**配合监管屏蔽请求。MetaMask 默认用 Infura，所以默认情况下你每一笔交易都过一家 ConsenSys 子公司的服务器。

Pocket Network、DRPC 这些去中心化 RPC 在做，但体验和可靠性还差一档，普通用户不会主动切换。

### 2.3 前端和域名是 Web 2

`uniswap.org`、`aave.com` 这些依然是 Web 2 域名 + Web 2 云托管。OFAC 一纸命令能立刻让这些前端瘫痪。

IPFS + ENS 前端（例如 `app.uniswap.eth.limo`）是**存在**的，但不是默认路径，普通用户根本不知道。当 Tornado Cash 被制裁时，它的官方前端立刻下线，但合约还在跑——因为合约在 Web 3，前端在 Web 2。

### 2.4 身份和支付入口仍被 Web 2 俘获

99% 的新用户从 Gmail + Coinbase KYC 进入加密世界。**入场即被俘获**——你已经完成了一轮完整的 Web 2 身份审核才拿到代币。

真正"钱包即身份"的原生用户几百万，对比全球几十亿互联网用户，这个比例仍然微不足道。

### 2.5 治理实际被资本控制

所谓的 DAO 投票被前 10 大代币持有者（往往是 VC + 内部人）绑定。Curve / Convex 战争本质上就是**"买下治理权"的军备竞赛**。

治理代币 = 证券，只是监管还没明说而已。SEC 对 Uniswap Labs 的调查已经在这条线索上推进。

## 三、分层心智模型：把加密堆栈画成五层

你可以把当前的加密世界画成这样的分层：

| 层级 | 代表组件 | 去中心化程度 |
|------|----------|-------------|
| 共识层 | Bitcoin / Ethereum 协议本身 | **真 Web 3** |
| 合约层 | 少数 immutable 协议（Uniswap V2、Liquity） | **真 Web 3**，但小众 |
| 应用层 | 大多数 DeFi 协议（Aave、Curve、Lido） | **Web 2.5**，多签 + DAO + proxy 升级 |
| 前端层 | `uniswap.org`、`aave.com` | **Web 2**，域名 + 云托管 |
| 入口层 | Coinbase KYC、Gmail 登录 | **Web 1.5**，KYC + 银行 |

你用的每一次 DeFi 交互其实是**穿过五层不同去中心化程度的堆栈**——底层的加密保证让你的资产不被没收，顶层的 Web 2 组件让你容易上手但也容易被拦截。

这就是为什么 Web 2.5 是**精确的描述**：它不是"一半去中心化"，而是"不同层达到不同程度的去中心化"。

## 四、产品分类全清单（本文的核心）

下面按我提出的三个维度——**谁能改协议 / 谁能冻你资产 / 出事能否自救**——给主流加密产品彻底归位。

### 4.1 真 Web 3（三层全部过关）

这类产品满足：**协议不可变或有硬 Timelock、无 admin 或 admin 权限极小、前端去中心化可选**。

| 产品 | 归位理由 |
|------|---------|
| **Bitcoin** | 全球 1 万多节点、没有"管理员"、没有公司、创始人消失。协议升级靠矿工+节点软共识，任何中心化权力都无法审查单笔交易。**唯一所有维度都过关的 Web 3**。 |
| **Ethereum（协议层）** | L1 本身无人能停，抗审查性经过 Tornado Cash 制裁实战验证。但 Ethereum **生态**（RPC、Lido 质押占比、多客户端分布）有中心化元素。**协议是 Web 3，生态是 Web 2.5**。 |
| **Monero / Zcash（协议）** | 原生隐私。Monero 更硬核（无可选透明）、Zcash 有公司影响但协议开源无 admin。隐私支付领域的真 Web 3。 |
| **Uniswap V2 合约** | 合约完全不可变、0 admin、部署 7 年没改过一行。但作为**产品**它的前端是 `uniswap.org`（Web 2）、UNI 归 Uniswap Labs 控制。**V2 合约是 Web 3，产品是 Web 2.5**。你用 Etherscan 的 Write Contract 直接调 V2 就是纯 Web 3 体验。 |
| **Liquity（LUSD）** | 借 ETH 铸 LUSD 稳定币、零 admin、零 governance 代币、IPFS 前端、匿名核心团队。完整三层都过关的极少数 DeFi，是"真 DeFi 活体样本"。 |
| **RAI / Reflexer** | 和 Liquity 类似的理念，零治理、浮动锚定的算法稳定币。小众但纯粹。 |
| **Tornado Cash（协议）** | 合约不可变、无 admin、OFAC 制裁都打不掉。**用实战证明了"协议级抗审查"是真的**。 |
| **Compound V2 合约层** | 合约本身不可变，但有 governance 可以调整参数。**合约是 Web 3，协议整体是 Web 2.5**。 |

> 这份清单加起来大概占加密行业市值的 60–70%（主要靠 BTC 和 ETH 拉），**但占日常使用量不到 10%**。大多数人日常用的是 Web 2.5 应用层。

### 4.2 Web 2.5（DeFi 的主战场）

这类产品：**资产自托管 + 治理/运营中心化 + 依赖中心化 infra**。

#### DEX 和借贷

| 产品 | 归位理由 |
|------|---------|
| **Uniswap（产品）** | V3/V4 有 admin、治理代币被 VC + Labs 实际控制、前端中心化、Labs 公司接受 SEC 调查后开始加白名单筛选。但底层合约不可改、LP 头寸夺不走。 |
| **Aave** | 借贷龙头，有 Proxy 可升级、多签 admin 能暂停市场、治理可以 freeze 资产（曾用过）。你的抵押品自托管，但协议参数完全由 DAO（事实上一小撮大户）控制。 |
| **Compound V3 / MakerDAO (DAI)** | DAI 是典型混合——抵押品一大部分是 USDC（Circle 能冻）、PSM 机制允许 MakerDAO 实际上做链上央行、DAO 投票被小圈子控制。**DAI 本质上是"半去中心化的链上美元"**。 |
| **Curve / Convex / Balancer / Yearn / GMX / Synthetix** | 这一批都是"核心合约可升级 + 多签 admin + 治理代币中心化 + 自托管用户资产"的标准 Web 2.5 配置。 |
| **dYdX** | 从以太坊迁到自己的 Cosmos 应用链，订单簿在链下（中心化撮合），结算上链。Web 2.5 里偏向 Web 2 那一头。 |

#### 质押和流动性

| 产品 | 归位理由 |
|------|---------|
| **Lido（stETH）** | 占以太坊质押 30%，是以太坊中心化风险的最大单点。有多签管理节点运营商白名单、能升级合约、LDO 被 VC 持有。但 stETH 可以自由转让交易。**Web 2.5 的典型教科书案例**。 |

#### L2 全家桶

| 产品 | 归位理由 |
|------|---------|
| **Arbitrum / Optimism / Base / zkSync / StarkNet / Scroll / Polygon zkEVM** | **所有 L2 现阶段都有中心化 sequencer（单点排序）+ admin 升级密钥**（能在任意时刻替换整个 L2 代码）。L2Beat 给它们打的"去中心化分数"普遍是 Stage 0 或 Stage 1，还没有一个到 Stage 2（真去中心化）。 |

#### 身份、钱包、社交

| 产品 | 归位理由 |
|------|---------|
| **ENS** | 真实合约自托管 ENS 名字没人能抢走，但根域名 `.eth` 的 controller 是一个 Safe 多签（ENS DAO 控制），理论上能改定价甚至 revoke。比普通 DNS 去中心化得多，但不是完全。 |
| **MetaMask / Rabby / Trust Wallet** | 自托管钱包，私钥你控制，但默认走中心化 RPC（Infura/Alchemy）、软件升级由公司推送、钱包公司能在前端加地址黑名单（MetaMask 已屏蔽过部分地址）。**"自托管但非抗审查"**。 |
| **Farcaster** | 号称去中心化社交，协议开源但实际 99% 流量过 Merkle Manufactory 的中心化 hub。**名义 Web 3 实际 Web 2.5**。 |

#### 稳定币分层

| 稳定币 | 分层 | 原因 |
|--------|------|------|
| **LUSD** | 真 Web 3 | 纯 ETH 抵押，零治理 |
| **DAI** | Web 2.5（混合） | 抵押品含大量 USDC，受 Circle 间接影响 |
| **GHO** | Web 2.5 | Aave 发行，治理中心化程度较高 |

### 4.3 Web 2（避雷区）

这类产品"只是跑在加密轨道上"，本质中心化。

#### 交易所

| 产品 | 归位理由 |
|------|---------|
| **Binance / Coinbase / OKX / Kraken / Bybit / Bitget** | 本质就是**数字银行**——你的币在他们手上，他们能冻能挪能破产。和支付宝一样性质，只是标的换成了加密资产。 |

#### 稳定币（发行方中心化）

| 产品 | 归位理由 |
|------|---------|
| **USDT / USDC** | 虽然在链上、虽然可自托管，但**发行方能单方面 blacklist 你的地址并 destroy 余额**。"谁能冻你资产"这层检查完全过不了。但注意——持有它们在链上比在银行账户里还是更自由（没人能查你身份），所以**实用性强于去中心化分数**。 |

#### 伪装成公链的中心化链

| 产品 | 归位理由 |
|------|---------|
| **Tron** | 超级代表（SR）系统只有 27 个节点、Justin Sun 事实上控制生态。**技术上叫区块链，治理上是私企**。 |
| **Ripple / XRP** | 公司保留大量代币、验证节点由 Ripple Labs 许可。典型 Web 2 伪装。 |
| **BSC (BNB Chain)** | 21 个验证者全是币安挑的。**币安背书的私有链**。 |
| **Solana（争议）** | 协议开源但验证者经济门槛极高、基金会影响巨大、网络宕机历史多次。**归为 Web 2.5，但偏 Web 2 那一侧**。 |

#### NFT 交易所

| 产品 | 归位理由 |
|------|---------|
| **OpenSea / Blur** | NFT 交易所的撮合和版税执行都在链下中心化服务器，曾审查过多个 NFT 系列。**NFT 资产是 Web 2.5，但交易平台是 Web 2**。 |

---

### 4.4 流量和市值的真实分布

三层堆叠下来，行业的真实现状是：

- **最硬核 Web 3** = Bitcoin + Ethereum 协议 + Monero + Liquity + Uniswap V2 合约 + Tornado Cash。清单加起来占市值 60–70%（主要靠 BTC 和 ETH），**但占日常使用量 < 10%**。
- **主流 DeFi 基本都是 Web 2.5**。你用 Aave、Uniswap、Lido、Curve、Arbitrum，就是在 Web 2.5 生态里。你的资产比在银行里自由得多（自托管），但协议本身并不是魔法。
- **Web 2 的 CEX 和稳定币** 实际承担 80% 以上的加密金融流量。Tether + Coinbase + Binance 三家加起来就是加密世界的 "JPMorgan + Chase + Visa"。

## 五、历史类比和时间线

### 5.1 Web 1.0 → 2.0 花了 10–15 年

1995–2010 这段历史值得回味：

- 最初几年大量人骂"互联网是骗局"——电商跑不起来、搜索难用、社交网络没有、连接速度慢
- Webvan、Pets.com 这些早期电商确实破产了
- 但基础设施（HTTP、DNS、TCP/IP）**一旦建成就不可逆**
- 之后 15 年各种"Web 2 应用"在上面长出来：Google、Facebook、YouTube、Netflix、Uber

### 5.2 Web 2.5 → Web 3 可能也是 15–20 年

我们大概在 2017–2018 进入 Web 2.5（ICO 泡沫 + DeFi Summer 萌芽）。现在是 2026 年，算 8 年进度。

按 Web 1→2 类比，还有 7–12 年到某种成熟态。

**但中间路径不会是线性的**：

- 会有熊市砸到 Web 2 退潮
- 会有政策反复
- 会有技术路线切换
- 会有新的中心化力量出现（比如现在的 Lido）

现在骂"Web3 是空气"就像 2002 年骂"电商是泡沫"——**你那时候是对的，但对的是时间点而不是方向**。

### 5.3 也有悲观剧本：Web 2.5 可能就是终点

诚实地讲，**Web 2.5 也可能永远到不了 Web 3**。三股力量都在把行业往 Web 2.5 拉，没有力量把它推向 Web 3：

1. **监管压力**让每个协议都加 admin（为自保）
2. **资本结构**让每个协议都发治理代币（为变现）
3. **用户体验要求**让每个产品都需要客服和救援（为增长）

30 年后回头看，"Web3 运动"的真实遗产可能不是去中心化，而是**给传统金融装上了数字轨道**——BlackRock 的 BUIDL、Circle 的 USDC、Coinbase 的 Base。这些会吞下市场，而 Liquity、Uniswap V2 这类真去中心化协议成为永远的"密码朋克保留地"。

### 5.4 我的判断：混合路径

**主流走向 "Web 2.5 + 监管合规化"，少数但关键的极端去中心化协议作为底线长期存在。**

大多数人不会直接用 Liquity，但 **Liquity 存在这件事本身约束了所有中心化玩家的作恶空间**。

> 这就像核武器不是为了使用而存在，是为了**使得不使用成为可能**。

## 六、对普通用户的实操建议

### 6.1 承认"Web 2.5 也有意义"是最诚实的入门姿态

这个认知让你：

- **不被乌托邦叙事忽悠**（FOMO 进骗局）
- **不被犬儒主义带偏**（错过真实进展）
- **分清楚每个产品在堆栈哪一层**（给风险正确定价）
- **把自己定位在合适的层次上参与**（投资、开发、使用）

### 6.2 三层使用策略

| 场景 | 推荐层级 | 原因 |
|------|---------|------|
| 长期持有 BTC/ETH | 真 Web 3（自托管） | 最大化抗审查性 |
| 日常 DeFi 交互 | Web 2.5（Aave、Uniswap） | 体验和效率最佳 |
| 法币出入金 | Web 2（CEX） | 合规通道 |
| **关键时刻（制裁、封号风险）** | **切换到真 Web 3** | 用 Liquity、IPFS 前端、Tornado Cash |

**你日常用 Web 2.5、心里知道 Web 3 的存在、关键时刻能切换到 Web 3 保护自己**——这是过去 5 年加密老手的真实操作模式，未来 5 年依然有效。

### 6.3 动手建议：体验一次真 Web 3

纸上谈兵没用。挑一件事今天就试一下：

- **选项 A**：打开 [Liquity](https://www.liquity.org/)，存一点 ETH，借一点 LUSD。走完流程你会意识到什么叫"零治理、零管理员"。
- **选项 B**：用 IPFS 前端访问 Uniswap V2（`app.uniswap.eth.limo` 或 Etherscan 直接 Write Contract 调合约）做一次 swap。对比 `uniswap.org` 你会意识到前端有多关键。
- **选项 C**：运行一个 Bitcoin 全节点（Umbrel 或 Start9 把门槛降得很低），你会第一次真实感受到"自己验证共识"。

走完一次，你对"真 Web 3 体验"就有了**具象认知**，再回头看 Web 2.5 产品的时候心里有杆秤。

## 七、结语

Web 2.5 这个框架最大的价值，是把"进展"解构成**可测量的维度**：

- 哪些功能已经真去中心化了
- 哪些还在 Web 2 模式
- 每年水位往哪里涨

这比"加密是革命 / 加密是骗局"的争论有用 100 倍——前者是**可操作的观察**，后者是**选边站的情绪**。

继续看 Etherscan、读合约、对比不同协议的治理结构，你会建立一种 **"去中心化鉴赏力"**——在未来 10 年的加密/金融交汇处，这会是稀缺能力，比任何具体的技术技能都值钱。
